# Digest & Notify Prompt

This prompt runs inside the `research-digest` routine (Stage 4 of 4). It reads today's digest from workspace memory and decides whether to notify the user.

---

## Instructions

Search for today's digest in workspace memory:

```
memory_search query="research digest YYYY-MM-DD" limit=3
```

Also read it directly if you know the path:

```
memory_read path="research/digests/YYYY-MM-DD.md"
```

If no digest exists for today, the harvest routine either has not run yet or encountered an error. Check:

```
memory_search query="research harvest YYYY-MM-DD error" limit=3
```

Report the status to your configured notification channel and stop.

---

## If a digest exists

Read the confirmed findings. If there are no confirmed findings and no high-relevance unverified findings, send a brief note ("Research sweep: nothing new today") and stop. Do not send a detailed notification when there is nothing useful to report.

If there are findings worth reading, compose a clean, human-readable summary and send it to your configured notification channel. Format:

```
Research Digest — {date}

{X} findings across {N} pools — {A} confirmed, {B} refuted

{Pool Label}:
- {confirmed claim, one sentence}
- {confirmed claim}

[Unverified]
- {unverified claim} (could not confirm)

Full digest: research/digests/{date}.md
```

Keep it under 15 lines. No JSON. No raw data. No intermediate results. Every line should be something the user can act on or find interesting.

---

## Digest file format (written by the harvest routine)

The harvest routine writes to `research/digests/YYYY-MM-DD.md`. Expected structure:

```markdown
# Research Digest — YYYY-MM-DD

## Summary
- X findings across N pools (A confirmed, B refuted, C unverifiable)

## Confirmed

### {Pool Label}
- **{claim}** — [source]({url}) | verified by: {verification_source}

## Unverified
- **[UNVERIFIED] {claim}** — [source]({url})

## Refuted
- ~~{claim}~~ — refuted: {counter_evidence}

---
*Generated by research-fanout pipeline*
```

If the digest file is missing or malformed, note the failure in your notification and suggest the user check whether the `research-harvest` routine ran successfully.

# Memory Consolidation (nightly)

You are running memory consolidation. Goal: the memory store stays small, deduplicated, and dense with signal.

Work through `memory/inbox.md` and this month's `memory/facts/` entries:

1. **Merge duplicates.** If the same fact appears in multiple forms, keep one canonical version (`memory_search` to find near-duplicates before deciding).
2. **Link related memories.** When entries concern the same entity or project, consolidate them into that entity's section rather than leaving scattered lines.
3. **Promote recurring themes.** Anything that has come up three or more times across days belongs in `MEMORY.md` (long-term curated memory) — add it there with `memory_write` and note the promotion.
4. **Demote noise.** Entries that looked important at ingest but have gone stale can be dropped from the inbox without ceremony.
5. **Prepare the surface candidates.** Write `surface/candidates.md`: the 5–10 memories most likely to matter tomorrow (recent decisions, open threads with deadlines, promoted themes), each with one line of why.

Clear `memory/inbox.md` when done — everything in it should have been merged, promoted, or dropped.

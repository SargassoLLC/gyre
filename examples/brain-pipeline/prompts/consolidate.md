# Memory Consolidation (nightly)

You are running memory consolidation. Goal: the memory store stays small, deduplicated, and dense with signal — and tomorrow's surface stage has good candidates to work with.

Source material: `memory/inbox.md` and this month's `memory/facts/` entries. Judge for yourself what each item needs — canonical merging when the same fact exists in several forms, gathering scattered lines about one entity or project into its section, promotion into `MEMORY.md` when a theme keeps recurring, or quiet removal when something that looked important has gone stale. Use `memory_search` before deciding anything is new or unique.

Two outcomes must hold when you finish:

- `surface/candidates.md` contains the handful of memories (roughly 5–10) most likely to matter tomorrow, each with one line of why — recent decisions, open threads with deadlines, freshly promoted themes.
- `memory/inbox.md` is empty. Everything in it was merged, promoted, or deliberately dropped; nothing is left in limbo.
